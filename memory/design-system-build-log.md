On July 6, 2026 we built a complete, build-ready design system for the founder's unnamed TCG website (price tracking, portfolio, trading, editorial) in `/Users/robinfox/Desktop/sandbox/testBuilderOS`. Two mirrored files, always updated together: `docs/design.md` (YAML tokens + prose, source of truth) and `docs/design.html` (self-contained, single-file, no-build-step style guide). Related: [[tcg-site-stack-decisions]].

**Foundation.** Visual language extracted from an external brand reference site by browsing it live: Japanese-material palette (Urushi ink #322014, Haku bronze gradient, Suna/Kami paper surfaces), full 50-950 light+dark ramps (44 tokens, every hex verified against the brand page), Tiempos Text/Geist pairing. Tiempos is paid, so we ran a 12-font visual bake-off (Google Fonts + Fontshare specimen page) and the founder picked Newsreader. The palette hero and tonal-ramp displays replicate the reference layout.

**Signature craft details.** Content cards have a corner "notch": the bottom-right corner is cut away with a ~20° diagonal edge into a shelf holding an arrow that expands to a "Read More" pill on hover. Took three iterations (90° step → too-round scoop → diagonal), final construction is a single SVG cap + filled strip because pieced-together CSS masks left seams; the failure modes are documented in design.md so builders don't repeat them. Rarity system: six game-agnostic tiers (common→secret) in "muted foil" colors mapping Magic, Pokémon, Yu-Gi-Oh, Warlord, and Sorcery; after an iteration board of five styling directions the founder chose tinted-wash badges (14% tier wash, 40% border, 70% tier-into-ink text) with a dot+label dense variant for tables.

**Scope built (~95 components/patterns, 77 stories, 21 sections):** buttons incl. loading/destructive/inverted, full form kit (combobox card search with JustTCG-shaped results, variant pickers, steppers, date picker, range slider, tag input, checkbox/radio/textarea, form-field pattern, auth card), chips/badges/freshness chip, card family (panel, content w/ notch, feature, card-frame, card grid tile with quick-add), navigation (72px header, dropdown, mega menu, mobile nav + sheet, tabs, pagination, breadcrumbs, user menu, notifications panel, wizard), editorial (article + sidebar widgets, blockquote/code/figure, accordion), marketing (hero, section header, pricing tiers, CTA band), Robinhood-style price components (price chart + portfolio hero as token-styled SVG mocks with the real Lightweight Charts theme config in design.md, sparklines, deltas, allocation bar, comparison chart, price matrix, position panel, holdings table with bulk selection + filter panel + sort control), community (profile header, star rating, trade offer, trade timeline, comments), settings (sections, API key row, danger zone), error page, onboarding checklist, feedback set (washed alerts, where the founder picked the tinted variant over left-border and underline; toast, banner, tooltip, skeleton, empty state), overlays (modal, drawer with add-card flow, popover card preview).

**System specs:** breakpoints/container/motion (150/300/600ms)/focus-ring/z-index scale, a full interaction & keyboard + ARIA spec, voice & formatting rules (two-decimal prices, one-decimal percents, relative-time ladder, "3.2% must never scan as 32%"), Lucide icon gallery with three-sizes rule, dark mode verified end to end (several contrast bugs found and fixed via sweep), mobile verified by actually resizing the browser (no horizontal scroll, tables collapse to cards).

**The Storybook chrome** (portfolio highlight): design.html is a hash-routed single file where every story has Preview / HTML / CSS / Docs tabs. HTML and CSS are extracted at runtime from the live DOM and stylesheet (matched rules only, media queries and keyframes included, token var() references preserved), and Docs fetches the live design.md and extracts the matching prose sentences + YAML token block. All four tabs are drift-proof by construction; a scripted audit confirms all 77 stories resolve docs.

**Verification style used throughout:** every change was screenshot-verified in Chrome (light + dark), the JustTCG API was curl-tested before designing around it (free tier: 100 req/day, daily priceHistory arrays), and real API numbers were used in chart demos.
