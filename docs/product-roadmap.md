## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** Phase 0 sets up everything so feature work is fast and clean.
3. **Magic moment first.** The first-ten-cards portfolio reveal works by the end of Phase 2. Everything after builds on it.
4. **Test as you go.** Every task's Notes end with a verification step. Do it before checking the box.
5. **Progressive enhancement.** Simplest working version first, polish layered on in Phase 5.
6. **Review before proceeding.** Push each completed phase as a PR (branch `phase-{N}/{slug}`) and let a review agent (CodeRabbit recommended) review before starting the next phase.
7. **Never block on the API.** All user-facing reads come from the Convex cache. JustTCG is touched only by actions and crons, within the daily budget. (Kiri-specific; see PRD § Technical Architecture.)
8. **The design system is law.** `docs/design.md` is complete and verified; implement components from it exactly rather than improvising. `docs/design.html` shows every component rendered, with HTML/CSS tabs.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a session:** Give your coding agent the phase prompt at the beginning of each phase.
2. **Read selectively:** Each phase lists its Reference sections: the specific parts of the PRD, vision doc, and design doc needed. Read only those, not the entire documents.
3. **Let it work:** The agent reads the roadmap, finds the first unchecked task, implements it, verifies it, and marks it complete.
4. **One session = one phase (ideally):** If you stop early, the agent resumes from the first unchecked task.
5. **Push a PR for review:** When a phase completes, push branch `phase-{N}/{slug}` and open a PR; let a review agent (CodeRabbit recommended) review before the next phase starts.
6. **Need more context?** If a task references a section not in the phase's list, read just that section on demand.

### Session Tips

- **Don't read everything:** `docs/design.html` is a rendered reference; the agent reads `docs/design.md` and only opens design.html's CSS via its code tabs when replicating a component.
- **Don't skip tasks:** Ordering resolves dependencies.
- **Verify after each task:** Every task's Notes end with a verification step; do it before checking the box.
- **Update the status line:** After completing tasks, update `**Status:** X/68 tasks complete` and `**Current Phase:**` in this file's header.
- **Budget awareness:** Any task touching JustTCG must respect the daily request budget in `syncState`; if a verification would burn requests, prefer fixtures.
